# Feature Specification: 姿勢スコア表示機能

**Feature Branch**: `001-posture-score`
**Created**: 2025-12-30
**Status**: Draft
**Input**: User description: "タスクバーに常駐、カメラに写った上半身の姿勢をスコア化し、アイコンの色でスコアを伝える機能"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - リアルタイム姿勢フィードバック (Priority: P1)

デスクワーク中のユーザーが、自分の姿勢状態を常にメニューバーアイコンの色で把握できる。良い姿勢を維持しているときは緑色、姿勢が崩れているときは黄色や赤色で表示され、意識的に姿勢を正すきっかけを得られる。

**Why this priority**: これがアプリの核心機能であり、ユーザーが最も価値を感じる部分。常時視覚的なフィードバックがなければ、アプリの存在意義がない。

**Independent Test**: カメラを有効にした状態でアプリを起動し、姿勢を変えながらメニューバーアイコンの色変化を確認することで、独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動していてカメラアクセスが許可されている状態, **When** ユーザーが良い姿勢（背筋を伸ばし、肩を開いた状態）を維持している, **Then** メニューバーアイコンが緑色で表示される
2. **Given** アプリが起動していてカメラアクセスが許可されている状態, **When** ユーザーの姿勢がやや崩れている（軽度の前傾や肩の丸まり）, **Then** メニューバーアイコンが黄色で表示される
3. **Given** アプリが起動していてカメラアクセスが許可されている状態, **When** ユーザーの姿勢が悪い（極度の前傾、首が前に出ている状態）, **Then** メニューバーアイコンが赤色で表示される

---

### User Story 2 - アプリの起動と常駐 (Priority: P1)

ユーザーがmacOSにログインすると、アプリがメニューバーに常駐し、バックグラウンドで姿勢監視を開始する。Dockにはアイコンが表示されず、メニューバーからのみアクセスできる。

**Why this priority**: メニューバー常駐はアプリの基本動作であり、これがなければ他の機能が成り立たない。

**Independent Test**: アプリを起動し、メニューバーにアイコンが表示されること、Dockに表示されないことを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリがインストールされている状態, **When** アプリを起動する, **Then** メニューバーにアプリアイコンが表示され、Dockにはアイコンが表示されない
2. **Given** アプリがメニューバーに常駐している状態, **When** メニューバーアイコンをクリックする, **Then** アプリのメニューまたは設定画面が表示される

---

### User Story 3 - カメラアクセス許可の管理 (Priority: P1)

初回起動時に、ユーザーはカメラへのアクセス許可を求められる。許可されると姿勢監視が開始され、拒否された場合は適切なメッセージとシステム設定への誘導が表示される。

**Why this priority**: カメラアクセスは姿勢検出の前提条件であり、適切な許可フローがなければアプリは機能しない。

**Independent Test**: 初回起動時の許可ダイアログ表示と、許可/拒否それぞれのケースでの動作を確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリを初めて起動した状態, **When** カメラアクセスの許可ダイアログが表示される, **Then** ユーザーが「許可」を選択すると姿勢監視が開始される
2. **Given** アプリを初めて起動した状態, **When** カメラアクセスの許可ダイアログで「拒否」を選択する, **Then** カメラアクセスが必要である旨のメッセージとシステム設定への誘導が表示される
3. **Given** カメラアクセスが拒否されている状態, **When** システム設定でカメラアクセスを許可に変更してアプリを再起動する, **Then** 姿勢監視が正常に開始される

---

### User Story 4 - アプリの終了 (Priority: P2)

ユーザーがメニューバーからアプリを終了できる。終了時にはカメラが適切に解放される。

**Why this priority**: 基本的な操作だが、メインの姿勢監視機能より優先度は低い。

**Independent Test**: メニューからアプリを終了し、プロセスが終了していることを確認することで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリがメニューバーに常駐している状態, **When** メニューから「終了」を選択する, **Then** アプリが終了し、カメラが解放される

---

### Edge Cases

- カメラが物理的に覆われている、または使用不可の場合はどうなるか？→ メニューバーアイコンがグレーになり、ツールチップで「カメラが利用できません」と表示
- 他のアプリがカメラを使用中の場合はどうなるか？→ カメラを共有利用できる場合は監視を継続、できない場合はグレーアイコンで待機状態を表示
- ユーザーが顔をカメラから外した（離席）場合はどうなるか？→ 人物が検出されない状態として、アイコンをグレーで表示
- 照明が極端に暗い/明るい場合はどうなるか？→ 検出精度が下がる可能性があるが、可能な限り判定を継続。判定不能な場合はグレーアイコン

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、macOSのメニューバーに常駐するステータスバーアプリケーションとして動作しなければならない
- **FR-002**: システムは、Dockにアイコンを表示してはならない（LSUIElement設定）
- **FR-003**: システムは、macOSのカメラアクセス許可を適切に要求し、ユーザーの選択に応じて動作しなければならない
- **FR-004**: システムは、カメラから取得した映像を使用して、ユーザーの上半身の姿勢をリアルタイムで分析しなければならない
- **FR-005**: システムは、姿勢分析の結果を0〜100のスコアとして算出しなければならない
- **FR-006**: システムは、姿勢スコアに応じてメニューバーアイコンの色を変更しなければならない（緑：良好、黄：注意、赤：改善が必要）
- **FR-007**: システムは、メニューバーアイコンをクリックした際にメニューを表示しなければならない
- **FR-008**: システムは、メニューから「終了」を選択することでアプリを終了できなければならない
- **FR-009**: システムは、カメラが利用できない場合（物理的遮蔽、他アプリ使用中、人物未検出）にグレーアイコンを表示しなければならない
- **FR-010**: システムは、プライバシー保護のため、カメラ映像を保存・送信してはならない（ローカル処理のみ）

### Key Entities

- **姿勢スコア（Posture Score）**: 0〜100の数値。ユーザーの現在の姿勢の良さを表す。高いほど良い姿勢。
- **スコア閾値（Score Thresholds）**: スコアを色に変換するための境界値。例：70以上=緑、40-69=黄、40未満=赤
- **監視状態（Monitoring State）**: アクティブ（正常監視中）、一時停止（カメラ利用不可）、無効（許可なし）の3状態

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが姿勢を変えてから2秒以内にアイコンの色が更新される
- **SC-002**: 良い姿勢を維持しているユーザーの90%以上が緑色アイコンとして正しく判定される
- **SC-003**: 明らかに悪い姿勢のユーザーの90%以上が赤色アイコンとして正しく判定される
- **SC-004**: アプリ起動から姿勢監視開始まで5秒以内に完了する
- **SC-005**: アプリが1日8時間稼働してもシステムリソース（メモリ、CPU）の著しい増加がない
- **SC-006**: カメラ利用不可時、3秒以内にグレーアイコンに切り替わる

## Assumptions

- macOS 14.6以上を対象とする
- 内蔵カメラまたはUSB接続のWebカメラが利用可能
- ユーザーはカメラの前に正対して座っていることを想定
- 上半身（頭、肩、背中の上部）がカメラに映っている状態を想定
- スコア閾値の具体的な値は実装・テスト段階で調整可能
