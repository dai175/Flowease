# Feature Specification: 姿勢アラート通知機能

**Feature Branch**: `006-posture-alert`
**Created**: 2026-01-11
**Status**: Draft
**Input**: Phase 1: 通知・アラート機能 - 悪い姿勢が続いたときにmacOS通知でユーザーに気づきを与える。累積ベースのトリガー（直近5分間の平均スコアが60以下で通知）、再通知制御（姿勢改善または最短通知間隔（デフォルト15分）経過で再通知可能）、設定可能な閾値・評価期間・通知間隔

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 悪い姿勢の通知を受け取る (Priority: P1)

ユーザーは作業中に姿勢が崩れていることに気づかないことが多い。一定時間悪い姿勢が続いた場合、macOSの通知で気づきを与え、姿勢を正すきっかけを提供する。

**Why this priority**: これが本機能の核心であり、通知がなければ機能として成立しない。ユーザーが能動的にスコアを見なくても、姿勢の悪化に気づける唯一の手段。

**Independent Test**: 姿勢スコアを意図的に60以下にして5分間維持し、通知が届くことを確認する。

**Acceptance Scenarios**:

1. **Given** キャリブレーション済みで姿勢監視中、**When** 直近5分間の平均スコアが60以下になる、**Then** macOS通知で「姿勢が崩れています」と通知される
2. **Given** 通知を受け取った後、**When** 姿勢を正して平均スコアが60を超える、**Then** 通知状態がリセットされ、再度悪化したときに通知される
3. **Given** 通知を受け取った後、**When** 姿勢を直さず平均スコアが閾値以下のまま最短通知間隔が経過する、**Then** リマインドとして再度通知が届く（姿勢改善または最短間隔経過のいずれかで再通知可能）

---

### User Story 2 - 通知設定をカスタマイズする (Priority: P2)

ユーザーによって「悪い姿勢」の基準や通知の頻度の好みは異なる。自分に合った閾値や通知間隔を設定できることで、煩わしすぎず、かつ効果的な通知を受け取れる。

**Why this priority**: 基本的な通知機能が動作した後に、ユーザー体験を向上させる機能。デフォルト値で十分なユーザーも多いため、P2とする。

**Independent Test**: 設定画面で閾値を変更し、変更後の閾値で通知が発火することを確認する。

**Acceptance Scenarios**:

1. **Given** 設定画面を開いた状態、**When** 閾値スコアを40に変更する、**Then** 平均スコアが40以下になったときに通知される
2. **Given** 設定画面を開いた状態、**When** 評価期間を3分に変更する、**Then** 直近3分間の平均で判定される
3. **Given** 設定画面を開いた状態、**When** 最短通知間隔を30分に変更する、**Then** 通知後30分間は再通知されない

---

### User Story 3 - 通知機能をオン/オフする (Priority: P2)

会議中やプレゼン中など、通知を受け取りたくない場面がある。通知機能を一時的に無効化できることで、状況に応じて使い分けられる。

**Why this priority**: 通知のオン/オフは設定カスタマイズの一部であり、基本機能の次に重要。

**Independent Test**: 通知をオフに設定し、悪い姿勢を続けても通知が来ないことを確認する。

**Acceptance Scenarios**:

1. **Given** 通知がオンの状態、**When** 通知をオフに設定する、**Then** 悪い姿勢が続いても通知は来ない
2. **Given** 通知がオフの状態、**When** 通知をオンに設定する、**Then** 悪い姿勢の検知と通知が再開される

---

### Edge Cases

- 姿勢監視が一時停止中（カメラ利用不可、顔未検出など）の場合、スコアが記録されないため通知判定は行わない
- アプリ起動直後、まだ評価期間分のスコアデータが蓄積されていない場合:
  - 評価期間の50%以上のデータがあれば、蓄積されたデータのみで平均を計算する
  - 評価期間の50%未満の場合は通知判定を行わない（例: 評価期間5分なら、最低2.5分以上のデータが必要）
- macOSの通知権限が許可されていない場合:
  - 通知機能がオンの場合のみ、ユーザーに権限を要求する
  - 通知機能がオフの場合は権限要求しない
- スコアが頻繁に閾値付近を行き来する場合でも、最短通知間隔により連続通知を防ぐ

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは姿勢スコアの時系列データを保持しなければならない（直近の評価期間分）
- **FR-002**: システムは設定された評価期間内の平均スコアを計算しなければならない
- **FR-003**: 平均スコアが閾値以下になった場合、システムはmacOS通知を送信しなければならない（通知テキストは日英両対応）
- **FR-004**: 通知送信後、以下のいずれかの条件を満たすまで再通知してはならない:
  - 姿勢が改善される（平均スコアが閾値を超える）、または
  - 最短通知間隔が経過する（リマインド通知）
- **FR-005**: 姿勢が改善された場合、通知状態をリセットし、再度悪化したときに通知できる状態に戻さなければならない
- **FR-006**: ユーザーは閾値スコア、評価期間、最短通知間隔を設定できなければならない
  - 閾値スコア: 20〜80の範囲
  - 評価期間: 1〜10分の範囲
  - 最短通知間隔: 5〜60分の範囲
- **FR-007**: ユーザーは通知機能のオン/オフを切り替えられなければならない
- **FR-008**: 設定値はアプリ再起動後も保持されなければならない
- **FR-009**: macOS通知権限が未許可かつ通知機能がオンの場合、システムは権限を要求しなければならない（通知オフ時は要求しない）
- **FR-010**: 姿勢監視が停止中の場合、スコア記録と通知判定をスキップしなければならない

### Key Entities

- **ScoreRecord**: 個々の姿勢スコアとそのタイムスタンプ
- **AlertSettings**: 通知に関するユーザー設定（閾値、評価期間、最短間隔、オン/オフ）
- **AlertState**: 通知の状態管理（最終通知時刻、姿勢改善フラグ）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 悪い姿勢が設定された評価期間続いた場合、通知が確実に届く（成功率100%）
- **SC-002**: 姿勢改善後に不要な通知が届かない（誤通知率0%）
- **SC-003**: 設定変更が即座に反映される（変更後1秒以内に新設定で判定開始）
- **SC-004**: ユーザーが設定画面で3タップ以内に任意の設定を変更できる
- **SC-005**: 通知の有効性について、ユーザーアンケート（5段階評価）で平均3.5以上を達成する

## Assumptions

- macOS 14.6以上の環境で動作する
- カメラ権限と通知権限はユーザーが許可することを前提とする
- 姿勢スコアは既存の姿勢分析機能から取得可能
- デフォルト設定値: 閾値60、評価期間5分、最短通知間隔15分
- 通知はmacOS標準のシステム通知を使用し、カスタムUIは不要
- 通知サウンドの設定はmacOSシステム設定に委ねる（アプリ内設定なし）
