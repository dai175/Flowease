# Feature Specification: 姿勢キャリブレーション機能

**Feature Branch**: `002-posture-calibration`
**Created**: 2026-01-01
**Status**: Draft
**Input**: User description: "姿勢検知が適切に行われていないため改善したいです。また、キャリブレーションも行う必要があると認識しています。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 良い姿勢のキャリブレーション (Priority: P1)

ユーザーが自分にとっての「良い姿勢」を基準として登録し、その基準に基づいて姿勢を評価してもらう。

現在の問題点として、全ユーザーに対して固定のしきい値を使用しているため、体格差や座り方の個人差を考慮できていない。キャリブレーション機能により、ユーザー個人の基準姿勢を記録し、そこからの逸脱を検知することで、より正確な姿勢評価が可能になる。

**Why this priority**: キャリブレーションは姿勢検知の精度向上の根本的な解決策であり、これなしでは他の改善も効果が限定的になる。

**Independent Test**: ユーザーがキャリブレーションを実行し、その後の姿勢スコアが個人の基準に基づいて計算されることを確認する。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** ユーザーがメニューから「キャリブレーション」を選択する、**Then** キャリブレーション画面が表示される
2. **Given** キャリブレーション画面が表示されている状態で、**When** ユーザーが良い姿勢を取り「基準姿勢を記録」ボタンをクリックする、**Then** 現在の姿勢が基準として保存される
3. **Given** 基準姿勢が保存されている状態で、**When** ユーザーが姿勢を崩す、**Then** 基準姿勢からの逸脱度合いに応じたスコアが表示される

---

### User Story 2 - キャリブレーションのリセット (Priority: P2)

ユーザーが座り方を変えた場合や、より良い姿勢を見つけた場合に、基準姿勢を再設定する。

**Why this priority**: キャリブレーションの再設定は、環境変化や姿勢改善に対応するために必要だが、初期キャリブレーションがなければ意味がない。

**Independent Test**: ユーザーがキャリブレーションをリセットし、新しい基準姿勢で評価が行われることを確認する。

**Acceptance Scenarios**:

1. **Given** 基準姿勢が既に設定されている状態で、**When** ユーザーが「キャリブレーションをリセット」を選択する、**Then** 現在の基準姿勢が削除される
2. **Given** 基準姿勢がリセットされた状態で、**When** ユーザーが新しい姿勢でキャリブレーションを実行する、**Then** 新しい姿勢が基準として保存される

---

### User Story 3 - キャリブレーション状態の視覚的表示 (Priority: P3)

ユーザーがキャリブレーションが完了しているかどうかを一目で確認できる。

**Why this priority**: UX向上のための機能であり、キャリブレーション自体が機能してから意味を持つ。

**Independent Test**: メニューバーまたはメニュー内でキャリブレーション状態が確認できることをテストする。

**Acceptance Scenarios**:

1. **Given** キャリブレーションが未実行の状態で、**When** ユーザーがメニューを開く、**Then** キャリブレーションが必要であることが明示される
2. **Given** キャリブレーションが完了している状態で、**When** ユーザーがメニューを開く、**Then** キャリブレーション完了状態が表示される

---

### Edge Cases

- カメラ映像で人物が検出されていない状態でキャリブレーションを試みた場合はどうなるか？ → エラーメッセージを表示し、人物が映るよう案内する
- キャリブレーション中に人物が画面から外れた場合はどうなるか？ → キャリブレーションを中断し、やり直しを促す
- 複数人がカメラに映っている状態でキャリブレーションを試みた場合はどうなるか？ → 一人だけが映るよう案内する（現在のVisionフレームワークは複数人検知をサポートしているが、本アプリでは単一ユーザーを想定）
- 極端に明るい/暗い環境でキャリブレーションを試みた場合はどうなるか？ → 信頼度が低い場合は警告を表示し、照明調整を促す
- キャリブレーション中に信頼度が低下し続けた場合はどうなるか？ → 信頼度0.7未満が連続1秒以上続いた場合は失敗とし、再試行を促す

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、ユーザーの現在の姿勢を「基準姿勢」として保存できなければならない
- **FR-002**: システムは、基準姿勢からの逸脱度合いを計測し、姿勢スコアを算出しなければならない
- **FR-003**: システムは、基準姿勢の各関節位置（首、肩、耳など）の相対的な位置関係を記録しなければならない
- **FR-004**: ユーザーは、保存された基準姿勢をリセットできなければならない
- **FR-005**: システムは、キャリブレーションの完了/未完了状態を永続化しなければならない（※基準姿勢の有無から導出。inProgress/failedは一時的な状態のため永続化対象外）
- **FR-006**: システムは、人物が検出されていない状態でのキャリブレーションを防止しなければならない
- **FR-007**: システムは、キャリブレーション時の姿勢の信頼度が0.7以上であることを確認しなければならない
- **FR-008**: システムは、キャリブレーション状態をアプリ再起動後も保持しなければならない（※基準姿勢データの永続化により実現。状態自体は永続化せず有無から導出）
- **FR-009**: システムは、キャリブレーション未実行時は既存の固定しきい値で姿勢評価を行い、キャリブレーション推奨メッセージを表示しなければならない
- **FR-010**: システムは、キャリブレーション中にプログレス表示（カウントダウンまたはプログレスバー）を表示しなければならない
- **FR-011**: システムは、姿勢スコアを既存の4項目（頭の傾き、肩のバランス、前傾、対称性）について基準姿勢からの逸脱度を計算し、0-100点で算出しなければならない
- **FR-012**: システムは、キャリブレーション中に信頼度0.7未満が連続1秒以上続いた場合は失敗とし、再試行を促さなければならない

### Key Entities

- **基準姿勢（ReferencePosture）**: ユーザーがキャリブレーションで設定した理想的な姿勢。3秒間（約90フレーム、最低30フレーム必要）の複数フレームを平均化して各関節の相対位置関係を保持する
- **姿勢逸脱度（PostureDeviation）**: 現在の姿勢が基準姿勢からどれだけ離れているかを示す指標
- **キャリブレーション状態（CalibrationState）**: キャリブレーションが完了しているか、実行中か、未実行かを表す状態

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: キャリブレーション完了後、ユーザーが「良い姿勢」をとった際に90点以上のスコアが表示される
- **SC-002**: キャリブレーション完了後、ユーザーが明らかに姿勢を崩した際に60点以下のスコアが表示される
- **SC-003**: キャリブレーションプロセスは30秒以内に完了できる
- **SC-004**: アプリ再起動後もキャリブレーション設定が維持される
- **SC-005**: 異なるユーザー（体格差のある）が各自キャリブレーションを行い、それぞれの基準で適切なスコアが算出される

## Clarifications

### Session 2026-01-01

- Q: キャリブレーション時の姿勢データ取得方法は？ → A: 複数フレーム平均（3秒間・約90フレームを平均化、最低30フレーム必要）
- Q: キャリブレーション時の信頼度しきい値は？ → A: 0.7以上（標準：バランス重視）
- Q: キャリブレーション未実行時の動作は？ → A: フォールバック動作（既存の固定しきい値で評価し、キャリブレーション推奨を表示）
- Q: キャリブレーション中のフィードバック表示は？ → A: プログレス表示（カウントダウンまたはプログレスバー）
- Q: スコア算出ロジックの方針は？ → A: 既存項目の差分評価（頭傾き・肩バランス・前傾・対称性を基準姿勢からの逸脱で再計算）

## Assumptions

- ユーザーは一人で本アプリを使用する（複数ユーザーの切り替え機能は今回スコープ外）
- 内蔵カメラまたは外付けWebカメラを使用する（カメラ位置・角度はユーザーが適切に設定する）
- キャリブレーション時、ユーザーは自分にとって快適で持続可能な「良い姿勢」を取る
- Visionフレームワークの姿勢検知の信頼度が一定以上であることをキャリブレーションの前提条件とする
