# Feature Specification: カメラ選択機能

**Feature Branch**: `005-camera-selection`
**Created**: 2026-01-08
**Status**: Draft
**Input**: User description: "カメラ選択機能"

## Clarifications

### Session 2026-01-08

- Q: 「デフォルトカメラ」の定義は？ → A: システムのデフォルトカメラ（OSが優先するカメラ）
- Q: 明示的に対象外とする項目は？ → A: カメラ設定（解像度・フレームレート等）の調整
- Q: カメラ切り替え中のUI状態は？ → A: 既存の「カメラ初期化中」状態を表示
- Q: カメラ利用不可時の通知方法は？ → A: メニュー内のステータス表示（アイコン変化など）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - カメラの選択 (Priority: P1)

外部Webカメラを使用しているユーザーが、内蔵カメラではなく外部カメラを姿勢モニタリングに使用したい。メニューバーの設定から使用するカメラを選択できる。

**Why this priority**: コア機能であり、この機能がないと外部カメラユーザーは適切なカメラを使用できない。

**Independent Test**: カメラ選択メニューからカメラを選択し、選択したカメラで姿勢モニタリングが動作することを確認できる。

**Acceptance Scenarios**:

1. **Given** 複数のカメラデバイスが接続されている状態、**When** ユーザーがアプリのメニューを開く、**Then** カメラ選択にアクセスできる（FR-008）
2. **Given** カメラ選択を開いた状態、**When** カメラリストを確認する、**Then** 利用可能なすべてのカメラがリストに表示され、現在選択中のカメラが視覚的に区別できる（FR-010）
3. **Given** カメラリストが表示されている状態、**When** ユーザーが別のカメラを選択する、**Then** 選択したカメラからの映像で姿勢モニタリングが開始される
4. **Given** カメラが選択された状態、**When** 後日アプリを起動する、**Then** 前回選択したカメラが自動的に使用される（設定の永続化）

---

### User Story 2 - カメラの切断・再接続時の動作 (Priority: P2)

外部カメラを使用中にカメラが切断された場合、適切なフィードバックを受け、再接続時にモニタリングを再開できる。

**Why this priority**: 外部カメラは物理的に着脱可能であり、切断時の適切なハンドリングはユーザー体験に重要。

**Independent Test**: 外部カメラを抜き差しして、切断時にモニタリングが一時停止し、再接続時に再開されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 外部カメラで姿勢モニタリング中、**When** カメラが切断される、**Then** モニタリングが一時停止し、カメラ切断のフィードバックが表示される
2. **Given** カメラが切断されてモニタリングが停止している状態、**When** カメラが再接続される、**Then** 自動的にモニタリングが再開される
3. **Given** 選択していたカメラが切断された状態、**When** 別のカメラが利用可能、**Then** 自動的に代替カメラに切り替わらず、ユーザーに選択を促す
4. **Given** カメラ選択画面を開いている状態、**When** 新しいカメラが接続される、**Then** カメラリストがリアルタイムで更新され新しいカメラが表示される（FR-005）
5. **Given** カメラ選択画面を開いている状態、**When** 接続中のカメラが切断される、**Then** カメラリストがリアルタイムで更新されそのカメラが非表示になる（FR-005）

---

### User Story 3 - カメラが1台のみの場合 (Priority: P3)

内蔵カメラのみを使用しているユーザーの場合、カメラ選択UIは表示されるが、選択肢は1つのみで自動的にそのカメラが使用される。

**Why this priority**: シングルカメラ環境でも機能は一貫して動作する必要があるが、複数カメラ環境が優先。

**Independent Test**: カメラが1台のみの環境で、カメラ選択メニューに1つのカメラのみ表示され、正常に動作することを確認できる。

**Acceptance Scenarios**:

1. **Given** カメラが1台のみ接続されている状態、**When** カメラ選択メニューを開く、**Then** 1台のカメラのみがリストに表示される
2. **Given** カメラが1台のみの状態、**When** アプリを起動する、**Then** 自動的にそのカメラが使用される

---

### Edge Cases

- カメラが0台の場合はどうするか？ → 既存の「カメラ利用不可」状態として処理される
- 選択済みのカメラが永続的に削除された場合は？ → 次回起動時にシステムのデフォルトカメラ（OSが優先するカメラ）にフォールバック
- カメラ権限がない状態でカメラ選択メニューを表示するか？ → カメラ権限取得後にのみカメラ選択を表示
- 同一モデル名のカメラが複数ある場合は？ → 同名のカメラは区別可能に表示される
- カメラ切り替え中の表示は？ → 既存の「カメラ初期化中」状態を表示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは接続されているすべてのビデオキャプチャデバイスを一覧表示できなければならない
- **FR-002**: ユーザーは一覧から使用するカメラを選択できなければならない
- **FR-003**: システムは選択されたカメラの設定を永続化し、アプリ再起動後も維持しなければならない
- **FR-004**: システムは選択されたカメラが利用不可の場合、メニュー内のステータス表示でユーザーに通知しなければならない
- **FR-005**: システムはカメラの接続・切断をリアルタイムで検知し、カメラリストを更新しなければならない
- **FR-006**: システムは選択されたカメラが切断された場合、モニタリングを一時停止しなければならない
- **FR-007**: システムは切断されたカメラが再接続された場合、自動的にモニタリングを再開しなければならない
- **FR-008**: カメラ選択UIはアプリのメニューからアクセス可能でなければならない
- **FR-009**: 各カメラはデバイス名で識別可能に表示されなければならない
- **FR-010**: 現在選択中のカメラは視覚的に区別できなければならない

### Key Entities

- **CameraDevice**: システムで利用可能なカメラデバイスを表す。属性として一意識別子、デバイス名、接続状態を持つ
- **CameraSelection**: ユーザーが選択したカメラの設定を表す。属性として選択されたカメラの識別子を持つ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは3クリック以内でカメラを切り替えられる
- **SC-002**: カメラ切り替え後、1秒以内に新しいカメラからの姿勢モニタリングが開始される
- **SC-003**: カメラの切断から2秒以内にユーザーに通知される
- **SC-004**: カメラの再接続から3秒以内にモニタリングが自動再開される
- **SC-005**: アプリ再起動後、前回選択したカメラが自動的に使用される（カメラが利用可能な場合）

## Dependencies

- 既存のカメラ権限管理機能（カメラ権限が許可されていることが前提）
- 既存の姿勢モニタリング機能（カメラ映像を受け取る仕組みが存在）

## Assumptions

- オペレーティングシステムがカメラデバイスの列挙と選択をサポートしている
- ユーザーは同時に1台のカメラのみを姿勢モニタリングに使用する
- カメラデバイスはシステムレベルで一意の識別子を持つ
- カメラの接続・切断イベントはシステムから通知される

## Out of Scope

- カメラ設定（解像度・フレームレート等）の調整機能
