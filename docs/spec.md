# デスクワーカー向け姿勢・健康管理アプリ 開発仕様書

## 1. プロジェクト概要

### 1.1 アプリ名
Flowease

### 1.2 コンセプト
**「デスクワーカーの体調管理パートナー」**

姿勢検知 + 症状別ストレッチガイド + 休憩リマインダーを統合し、デスクワークによる頭痛・肩こり・腰痛を予防・改善するMacデスクトップアプリ。

### 1.3 開発の動機
- 開発者自身がデスクワークによる頭痛、姿勢の悪さ、肩・腰の痛みに悩んでいる
- 既存アプリは姿勢検知のみ、または休憩リマインダーのみで、統合的なソリューションがない
- 自分の問題を解決しながら、技術を学ぶ

### 1.4 ターゲットユーザー
- デスクワーク中心のプログラマー、デザイナー、ライターなど
- 頭痛、肩こり、腰痛に悩んでいる人
- Macユーザー

## 2. 機能要件

### 2.1 MVP（最小限の製品）

| 機能 | 説明 | 優先度 |
|------|------|--------|
| **姿勢検知** | Macのカメラで前かがみ・首の傾きを検知し、通知 | 必須 |
| **休憩リマインダー** | 30〜60分ごとに休憩を促す通知 | 必須 |
| **ストレッチガイド** | 休憩時に簡単なストレッチをアニメーションで表示 | 必須 |
| **メニューバー常駐** | 邪魔にならないUI、姿勢状態を色で表示 | 必須 |

### 2.2 将来の機能（v2以降）

| 機能 | 説明 |
|------|------|
| **症状トラッキング** | 今日の調子（頭痛、肩こり、腰痛など）を記録 |
| **症状別ストレッチ提案** | 「頭痛がひどい」→ 首・肩のストレッチを重点的に |
| **統計・可視化** | 姿勢スコア、症状の傾向をグラフ表示 |
| **ポモドーロタイマー連携** | 作業と休憩のサイクル管理 |
| **省電力モード** | 常時監視ではなく定期スナップショット |

## 5. 技術詳細

### 5.1 姿勢検知の仕組み

Vision Frameworkの`VNDetectHumanBodyPoseRequest`を使用して、以下のポイントを検出：

```
検出可能なポイント（上半身）：
- 鼻
- 耳（左右）
- 目（左右）
- 首
- 肩（左右）
- 肘（左右）
- 手首（左右）
```

#### 判定ロジック

| 判定項目 | 検知方法 |
|---------|---------|
| 前かがみ（猫背） | 鼻・首の位置が基準より前に出ている |
| 首の傾き | 耳の左右の高さの差 |
| 肩のすくみ | 肩の位置が基準より上がっている |
| 画面との距離 | 顔のサイズの変化 |

#### サンプルコード（概念）

```swift
import Vision

func detectPosture(from image: CGImage) {
    let request = VNDetectHumanBodyPoseRequest { request, error in
        guard let observation = request.results?.first as? VNHumanBodyPoseObservation else { return }
        
        let nose = try? observation.recognizedPoint(.nose)
        let neck = try? observation.recognizedPoint(.neck)
        
        // 前かがみ判定
        if let nose = nose, let neck = neck {
            let forwardLean = nose.location.x - neck.location.x
            if forwardLean > threshold {
                showPostureWarning()
            }
        }
    }
    
    let handler = VNImageRequestHandler(cgImage: image)
    try? handler.perform([request])
}
```

### 5.3 ストレッチガイド

#### 収録予定のストレッチ（MVP）

| カテゴリ | ストレッチ名 | 所要時間 |
|---------|-------------|---------|
| 首 | 首回し | 30秒 |
| 首 | 首筋伸ばし | 30秒 |
| 肩 | 肩回し | 30秒 |
| 肩 | 肩甲骨寄せ | 30秒 |
| 腰 | 腰ひねり | 30秒 |
| 全身 | 背伸び | 15秒 |

#### 表示形式
- シンプルなアニメーション（Lottie or SwiftUI Animation）
- タイマー付き
- 音声ガイドは後のバージョンで検討

## 6. UI/UX設計

### 6.1 デザイン原則

1. **邪魔にならない** - 作業の妨げにならないメニューバー常駐
2. **シンプル** - 必要最小限の情報表示
3. **フィードバック** - 姿勢状態を色で直感的に表示
4. **達成感** - ストレッチ完了時のポジティブなフィードバック

### 6.2 カラースキーム

| 状態 | 色 | 意味 |
|------|-----|------|
| 良い姿勢 | 🟢 緑 | 問題なし |
| 注意 | 🟡 黄 | 少し姿勢が崩れている |
| 悪い姿勢 | 🔴 赤 | 姿勢を直す必要あり |

### 6.3 通知の種類

| 種類 | トリガー | 内容 |
|------|---------|------|
| 姿勢警告 | 悪い姿勢が一定時間続く | 「姿勢を直しましょう」 |
| 休憩リマインダー | 設定時間経過 | 「休憩の時間です」 |
| ストレッチ提案 | 休憩時 | 「肩回しストレッチをしませんか？」 |

## 9. 注意事項

### 9.1 プライバシー

- カメラ映像はローカルでのみ処理
- 映像は保存・送信しない
- Info.plistにカメラ使用理由を明記

### 9.2 パフォーマンス

- 常時カメラ監視は電力消費が大きい（競合アプリで19Wの報告あり）
- 定期スナップショット方式を検討
- バックグラウンド動作時の最適化

### 9.3 ユーザー体験

- 通知が多すぎると無視される
- 姿勢判定の閾値は調整可能に
- 初回キャリブレーションを簡単に
